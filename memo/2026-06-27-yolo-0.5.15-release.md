# yolo 0.5.15 patch release

Date: 2026-06-27 14:07 JST

## Scope

Patch release for the recovery-loop hardening introduced after v0.5.14.

## Included changes

- Bound Codex app-server self-heal retries with exponential backoff.
- Avoid destructive app-server replacement when an existing app-server is
  reachable or can be adopted.
- Clear inferred thread ownership when it conflicts with explicit
  `resume THREAD_ID` clients.
- Clear duplicate inferred thread ownership so cwd-based matching does not
  make multiple yolo clients own the same Codex thread.
- Preserve YOLO-mode permissions refresh for live resume clients.
